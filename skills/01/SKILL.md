# 시작하기: 첫 걸음

> Claude Code가 처음이라면 여기서 시작하세요.

## 터미널이란?

컴퓨터와 글자로 대화하는 창입니다. 마우스로 클릭하는 대신, 글자를 입력해서 명령합니다.
Claude Code는 이 창 안에서 동작합니다.

## 화면 기호 읽는 법

| 기호 | 의미 |
|------|------|
| `>` | 여기에 입력하세요 (프롬프트) |
| `◐` | Claude가 생각/작업 중입니다 (기다려주세요) |
| `✓` | 완료되었습니다 |
| `✗` | 실패했습니다 |
| `y/n` | 허용(y) 또는 거부(n)를 선택하세요 |

## 가장 먼저 해볼 것

1. Claude Code를 열고 **"안녕, 넌 뭘 할 수 있어?"** 라고 입력해 보세요
2. Claude가 대답합니다 — 이것이 기본 대화입니다
3. **"이 폴더에 어떤 파일이 있어?"** 라고 물어보세요
4. Claude가 현재 폴더를 탐색해서 알려줍니다

## 안전한가요?

- Claude는 기본적으로 파일을 수정하기 전에 **항상 물어봅니다**
- "허용하시겠습니까?" 질문에 `y`(yes) 또는 `n`(no)으로 대답합니다
- 실수했다면 레슨 02의 되돌리기(`/rewind`)를 사용하면 됩니다
- 무엇을 입력해야 할지 모르겠다면, 한국어로 자연스럽게 말하면 됩니다

## 자주 쓰는 명령어

`/`를 입력하면 명령어 목록이 나타납니다. 가장 자주 쓰는 명령어들입니다:

| 명령어 | 설명 | 언제 쓸까? |
|--------|------|-----------|
| `/help` | 도움말 보기 | 뭘 해야 할지 모를 때 |
| `/clear` | 대화 내용 지우기 | 새로 시작하고 싶을 때 |
| `/model` | AI 모델 변경 | 더 빠르거나 똑똑한 모델로 전환 |
| `/cost` | 사용량 확인 | 얼마나 썼는지 확인 |
| `/rewind` | 되돌리기 | 방금 한 작업 취소 |
| `/resume` | 이전 대화 이어하기 | 어제 하던 작업 계속 |
| `/compact` | 대화 내용 압축 | 대화가 길어졌을 때 정리 |
| `/diff` | 변경 내용 확인 | 뭐가 바뀌었는지 보기 |
| `/export` | 대화 내보내기 | 대화 내용을 텍스트로 저장 |
| `/doctor` | 설치 상태 점검 | 뭔가 안 될 때 진단 |

## 팁

- 명령어를 외울 필요 없습니다 — `/help`로 언제든 도움 받기
- `Esc` 키를 누르면 Claude가 하던 작업을 중단합니다

**꼭 기억하세요:** `Esc` = 중단, `Esc Esc` = 되돌리기. 모르겠으면 `n`(거부)이 안전합니다.

> `0` = 목차 | 레슨 번호 입력 = 해당 레슨으로 이동 | **"완료"** = 이 레슨 완료 표시

사용자가 "완료"라고 입력하면 아래 명령어를 실행하고 "레슨 1 완료! 0을 입력하면 목차에서 진척률을 확인할 수 있습니다."라고 안내하세요:
```bash
mkdir -p ~/.cc-start-kr && echo 1 >> ~/.cc-start-kr/completed && sort -un -o ~/.cc-start-kr/completed ~/.cc-start-kr/completed
```
