# 무엇이든 되돌리기

> 레슨 1에서 Claude와 대화하는 법을 배웠습니다. 이제 실수해도 되돌릴 수 있다는 걸 배워볼까요?

## 이게 뭔가요?

게임의 세이브 포인트처럼, Claude는 파일을 수정하기 전에 자동으로 저장합니다.
잘못되었다면 언제든 이전 상태로 되돌릴 수 있습니다. 실수해도 괜찮아요!

## 핵심 개념

되돌리기를 실행하는 방법은 두 가지입니다:

- **`Esc Esc`** — 키보드에서 Esc를 빠르게 두 번 누르기 (가장 빠른 방법)
- **`/rewind`** — 프롬프트에 `/rewind`를 입력하기 (같은 기능)

두 방법 모두 되돌리기 화면이 열리고, 파일과 대화를 이전 상태로 되돌릴 수 있습니다.

## 사용 예시

상황: Claude에게 보고서를 수정해달라고 했는데, 결과가 마음에 안 듭니다.

```
> 보고서.txt에서 요약 부분을 더 짧게 줄여줘
  ✓ 보고서.txt 수정 완료              ← Claude가 파일을 수정했어요

  (읽어보니 너무 많이 줄여서 내용이 빠졌다!)

  Esc Esc (빠르게 두 번 누르기)        ← 되돌리기 시작!

  되돌릴 지점:
    ❯ 보고서.txt 수정 전              ← 이 지점으로 되돌리기

  ✓ 보고서.txt 복원됨                 ← 원래대로 돌아왔어요!

> 요약은 유지하되 핵심 수치는 남겨줘    ← 다른 방식으로 다시 요청
```

**흐름 정리:**
1. Claude에게 작업을 요청했어요
2. 결과가 마음에 안 들었어요
3. `Esc Esc`를 눌러서 수정 전으로 되돌렸어요
4. 다른 방식으로 다시 요청했어요

## 되돌리기 vs 대화 지우기 vs 분기

| 방법 | 언제 쓸까? | 파일은? |
|------|-----------|--------|
| `Esc Esc` (되돌리기) | 방금 한 수정을 취소하고 싶을 때 | 파일도 원래대로 복원 |
| `/clear` (대화 지우기) | 대화를 처음부터 새로 시작하고 싶을 때 | 파일은 그대로 유지 |
| `/branch` (분기) | 두 가지 방법을 비교해보고 싶을 때 | 각 분기별로 따로 관리 |

**꼭 기억하세요:** `Esc` = 중단, `Esc Esc` = 되돌리기. 모르겠으면 `n`(거부)이 안전합니다.

> `0` = 목차 | 레슨 번호 입력 = 해당 레슨으로 이동 | **"완료"** = 이 레슨 완료 표시
